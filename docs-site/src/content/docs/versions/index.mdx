## How to read versioned docs

1. Start with the versioned page that matches your pricing year or release.
2. Use the governance pages for stable repo policy and implementation rules.
3. Treat the versioned docs as the public contract for that year, not as a
   substitute for source archive provenance.

## Versioning guidance

- Keep versioned pages short and explicit.
- Link back to the canonical governance pages instead of duplicating rules.
- Add new version entries only when the project actually publishes a new
  calculator release or pricing year.
