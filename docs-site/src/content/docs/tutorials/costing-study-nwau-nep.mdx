This tutorial walks through the calculation of **notional equivalent funding (NEP)** from admitted patient activity data using the `nwau_py` package. The result represents what a jurisdiction *would have been paid* under a given pricing year's national efficient price (NEP).

<Aside type="caution" title="Synthetic data only">
  All data used in this tutorial is synthetic and for illustration purposes only. Do not use these outputs for policy decisions, jurisdictional submissions, or financial reporting.
</Aside>

## Prerequisites

- Python 3.10+
- `nwau_py >= 0.6.0` installed
- Basic familiarity with activity-based funding (ABF) concepts

## 1. Load the package and data

```python
import pandas as pd
from nwau_py.costing import CostingEngine
from nwau_py.loaders import load_sample_activity

engine = CostingEngine(pricing_year=2025)
activity = load_sample_activity(source="synthetic_2025")
```

The `load_sample_activity` helper returns a DataFrame with columns for episode-level DRG, length of stay, separation mode, and patient demographic indicators.

## 2. Price each episode

```python
priced = engine.price_activity(
    activity,
    price_type="nep",           # National Efficient Price
    include_weights=True,
    include_price=True,
)

priced[["episode_id", "drg", "weight", "nep_price"]].head()
```

The engine applies the NEP pricer table for the selected year, returning a weight per DRG and the corresponding episode-level price.

## 3. Aggregate to notional funding

```python
notional_funding = (
    priced.groupby(["jurisdiction", "stream"])["nep_price"]
    .agg(["count", "sum", "mean"])
    .round(2)
)

notional_funding.columns = ["episodes", "total_nep", "mean_nep_per_episode"]
notional_funding
```

This produces a jurisdiction-by-stream table showing the **total notional funding** that would be implied at the national price.

## 4. Export for downstream use

```python
notional_funding.to_csv("notional_funding_by_stream.csv")
```
