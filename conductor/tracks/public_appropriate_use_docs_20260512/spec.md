# Specification: Public Appropriate-Use Documentation

## Overview
Add public-facing documentation that prevents overclaiming and helps users
understand appropriate use. The docs should clearly separate validated behavior,
experimental behavior, roadmap-only surfaces, policy caveats, licensing
boundaries, and non-endorsement by IHACPA unless that status changes.

## Functional Requirements
- Add an appropriate-use page.
- Add a validation-status and support-claims page.
- Add source/licensing caveats for IHACPA materials and restricted
  classification products.
- Link public docs from README and docs navigation.
- Include caveats for funding, pricing, costing studies, and policy
  interpretation.

## Acceptance Criteria
- Users can tell what is validated, experimental, future-only, or locally
  supplied.
- Docs state that the project cites IHACPA sources but is not official IHACPA
  software unless separately endorsed.
- Docs clarify that NWAU/NEP calculations are analytical aids and not complete
  funding determinations.
